# Microsoft Learn: Progress Bars (Win32 apps)

Source URL: https://learn.microsoft.com/en-us/windows/win32/uxguide/progress-bars
Accessed: 2026-02-22

## Why this matters for “stalled job” triage

- Time remaining is only helpful when it is accurate; misleading/“way off” estimates are not helpful.
- A progress indicator should make it clear when progress is (and is not) being made, so users don’t wait indefinitely on a task that won’t complete.

## Relevant excerpts (short)

- “Provide a time remaining estimate if you can do so accurately. Otherwise, don't provide one. Inaccurate time remaining estimates aren't helpful.”
- “Clearly indicate lack of progress.”
- “You don't want users to wait indefinitely for an operation that is never going to complete.”
