# RFC 9110: "HTTP Semantics" (June 2022) - 202 (Accepted)

- Type: IETF Standards Track RFC (STD 97)
- Authors: R. Fielding (Ed.), M. Nottingham (Ed.), J. Reschke (Ed.)
- URL: https://www.rfc-editor.org/rfc/rfc9110.html
- Accessed: 2026-02-22

## Why this source matters for this skill

RFC 9110 defines the meaning of HTTP 202 (Accepted): the request can be acknowledged as accepted for processing even though the processing is not complete. This is the cleanest, protocol-level analogy for a "runtime estimate / accepted submission" UI that still doesn't guarantee completion.

## Relevant excerpt (verbatim, short)

- "The 202 (Accepted) status code indicates that the request has been accepted for processing, but the processing has not been completed."
